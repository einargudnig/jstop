**Key decisions for this plan:**
- MenuBarExtra (macOS 13+) chosen over NSStatusItem — SwiftUI-native, cleaner code
- App Sandbox DISABLED — required for ps/lsof process enumeration in Phase 2; set correctly now
- LSUIElement=YES in Info.plist — prevents Dock icon from launch (more reliable than runtime activation policy)
- Deployment target: macOS 13.0 — minimum for MenuBarExtra API
- XcodeGen for project creation — only reliable CLI-based .xcodeproj generation

<task type="auto">
  <name>Task 1: Scaffold Xcode project with XcodeGen</name>
  <files>project.yml, NodeProcesses/Assets.xcassets/AppIcon.appiconset/Contents.json, NodeProcesses/Assets.xcassets/Contents.json, NodeProcesses/Assets.xcassets/AccentColor.colorset/Contents.json</files>
  <action>
1. Check if xcodegen is installed: `which xcodegen || brew install xcodegen`

2. Create `project.yml` at project root:

```yaml
name: NodeProcesses
options:
  bundleIdPrefix: com.nodeprocesses
  minimumXcodeVersion: "14.0"
  deploymentTarget:
    macOS: "13.0"

settings:
  base:
    SWIFT_VERSION: "5.9"
    MACOSX_DEPLOYMENT_TARGET: "13.0"
    ENABLE_HARDENED_RUNTIME: NO

targets:
  NodeProcesses:
    type: application
    platform: macOS
    sources:
      - NodeProcesses
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.nodeprocesses.NodeProcesses
        INFOPLIST_FILE: NodeProcesses/Info.plist
        CODE_SIGN_ENTITLEMENTS: NodeProcesses/NodeProcesses.entitlements
        CODE_SIGN_IDENTITY: "-"
        CODE_SIGNING_REQUIRED: "NO"
        CODE_SIGNING_ALLOWED: "NO"
    scheme:
      testTargets: []
```

3. Create minimal Assets.xcassets structure:
   - `NodeProcesses/Assets.xcassets/Contents.json` with `{"info":{"author":"xcode","version":1}}`
   - `NodeProcesses/Assets.xcassets/AccentColor.colorset/Contents.json` with standard accent color JSON
   - `NodeProcesses/Assets.xcassets/AppIcon.appiconset/Contents.json` with empty app icon set JSON

4. Run: `xcodegen generate`

5. Verify the project was created: `ls NodeProcesses.xcodeproj/project.pbxproj`

<task type="auto">
  <name>Task 2: Configure menu bar agent settings</name>
  <files>NodeProcesses/Info.plist, NodeProcesses/NodeProcesses.entitlements</files>
  <action>
1. Create `NodeProcesses/Info.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleExecutable</key>
    <string>$(EXECUTABLE_NAME)</string>
    <key>CFBundleIdentifier</key>
    <string>$(PRODUCT_BUNDLE_IDENTIFIER)</string>
    <key>CFBundleInfoDictionaryVersion</key>
    <string>6.0</string>
    <key>CFBundleName</key>
    <string>$(PRODUCT_NAME)</string>
    <key>CFBundlePackageType</key>
    <string>$(PRODUCT_BUNDLE_PACKAGE_TYPE)</string>
    <key>CFBundleShortVersionString</key>
    <string>1.0</string>
    <key>CFBundleVersion</key>
    <string>1</string>
    <key>LSUIElement</key>
    <true/>
    <key>NSHighResolutionCapable</key>
    <true/>
    <key>NSPrincipalClass</key>
    <string>NSApplication</string>
</dict>
</plist>
```

Key: `LSUIElement = true` makes this a background app — no Dock icon, no app switcher entry.

2. Create `NodeProcesses/NodeProcesses.entitlements`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.app-sandbox</key>
    <false/>
</dict>
</plist>
```

App Sandbox MUST be false — sandboxed apps cannot enumerate other processes or run lsof.

3. Create a placeholder Swift file so the target has at least one source file:
   Create `NodeProcesses/NodeProcessesApp.swift` with just a comment `// NodeProcesses — placeholder, replaced in plan 01-02`
   (XcodeGen requires at least one source file to create a valid target)

4. Run `xcodebuild -project NodeProcesses.xcodeproj -scheme NodeProcesses -configuration Debug build CODE_SIGN_IDENTITY="-" CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO 2>&1 | tail -5`

  <done>xcodebuild reports BUILD SUCCEEDED. NodeProcesses.xcodeproj is buildable with LSUIElement and sandbox disabled.</done>
